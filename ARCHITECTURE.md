# Architecture

This is a design-document-only repository. There is no code to architect or deploy. The "architecture" here is the conceptual framework for reasoning about coding swarm autonomy levels.

## Conceptual dimensions

- **Autonomy spectrum**: user-driven → semi-autonomous → fully-autonomous
- **Coordination models**: how multiple agents divide, sequence, and integrate work
- **Feedback loops**: how the swarm learns from execution outcomes

See `docs/architecture/` for detailed conceptual models, C4-style diagrams of swarm topologies, and DDD bounded context maps for the domain of coding swarm orchestration.
